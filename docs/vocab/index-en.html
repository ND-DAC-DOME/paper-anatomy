Paper Anatomy Vocabulary (PAX): Description back to ToC

The Paper Anatomy Vocabulary (PAX): a small vocabulary complementing DoCO, DEO, FaBiO, the W3C Web Annotation model and PROV-O for describing the anatomy of a scientific paper — its structural, rhetorical, spatial and part–whole composition. It covers only what those vocabularies do not: AI-extracted figure semantics (figure type as a SKOS scheme, chart data payloads), the distinction between a page's sequence position and its printed page label, spatial-anchoring conveniences (an application profile over oa:ResourceSelection, deliberately not a conforming Web Annotation), and document components absent from DoCO (running page headers/footers, printed page-number marks, code blocks). Design rule: a term is added here ONLY when no suitable term exists in the reused community vocabularies. See the schema documentation for the full graph design. Status: testing — not yet registered at w3id.org; under review before publication.

regionop back to ToC or Object Property ToC

IRI: https://w3id.org/paper-anatomy/vocab#region

Links a document element to an oa:ResourceSelection (source + FragmentSelector with a Media Fragments xywh value) anchoring it to a rectangular region of a page image. This is a compact APPLICATION PROFILE reusing Web Annotation vocabulary components; it is deliberately NOT a conforming oa:Annotation and will not be consumed as-is by generic annotation clients. A lossless transform to conforming Annotations is possible. One element may carry several regions when it spans columns or pages. Coordinates are in the pixel space whose dimensions are given by pax:pageImageWidth/Height on the page.
Is defined by
https://w3id.org/paper-anatomy/vocab
has range

chart datadp back to ToC or Data Property ToC

IRI: https://w3id.org/paper-anatomy/vocab#chartData

The structured data extracted from a chart-type figure (axes, series, data points, error bars), embedded verbatim as a JSON literal (rdf:JSON, JSON-LD 1.1 '@json'). Immutable extraction evidence — payload, not graph structure. No rdfs:range is declared: rdf:JSON is outside the OWL 2 datatype map, and a range axiom would push the vocabulary out of the DL profile. Normalized point nodes may complement (not replace) this when cross-paper SPARQL querying, per-point provenance, or unit alignment is required.
Is defined by
https://w3id.org/paper-anatomy/vocab
