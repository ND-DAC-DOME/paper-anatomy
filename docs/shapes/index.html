PAX SHACL validation profiles

Namespace https://w3id.org/paper-anatomy/shapes# (prefix paxs) — one namespace across all three profile documents. Severity is part of the contract:

ProfileSeverityPurpose
core.ttlsh:Violation Structural requirements — a graph failing these is not a valid paper-structure KG. Conformance to PAX = zero core violations. Includes paxs:ChartDataShape (enforces the rdf:JSON datatype the OWL layer deliberately leaves unstated).
recommended.ttlsh:Warning Interoperability + consistency of materialized projections: pax:matter must agree with the containing matter node; pax:pageStart/End with actual containment; po:contains ≡ dcterms:hasPart on boxes.
diagnostic.ttlsh:Info Data-quality signals (text-less paragraphs/captions, …) — surfaced honestly, never blocking.

Run (from the repository root; validates the shipped example through all three profiles plus term-coverage and region-bounds checks):

uv run --with rdflib --with pyshacl python validate.py

Scoring (IoU, coordinate tolerances, reading-order distance) is deliberately not here — SHACL severity has no tolerance semantics; those are evaluation-engine metrics. See the vocabulary landing page for the full status and conformance statement.
